Notas:

Cada vez que un contendor se ejecuta, en realidad lo que ejecuta es un proceso del sistema operativo. Este proceso se le conoce como `Main process`.

`Main process` determina la vida del contenedor, un contendor corre siempre y cuando su proceso principal este corriendo.

`Sub process` => Un contenedor puede tener o lanzar procesos alternos al main process, si estos fallan el contenedor va a seguir encedido a menos que falle el `Main process`.

Lo que necesitamos hacer es correr un proceso que no sea `bin\bash` y que no haga `exit`, para hacer eso vamos a correr un nuevo contenedor:
ðŸ‘‰ docker run --name alwaysup -d ubuntu tail -f /dev/null
